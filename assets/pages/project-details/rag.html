RAG System with Guardrails

Retrieval-Augmented Generation (RAG) system with guardrails features using FastAPI, FAISS, and guardrails.

The system supports hybrid retrieval (BM25 + FAISS) with Reciprocal Rank Fusion (RRF) and Maximal Marginal Relevance (MMR), answer generation with citations, and a self-correction mechanism (WIP).

The key components and techniques used in this project include:

  • FastAPI for building the web service with `POST /retrieve` and `POST /answer` endpoints. and Pydantic for data validation and serialization.
  • Data ingestion from Wikivoyage, heading-aware chunking, SentenceTransformer embeddings, FAISS indexing (efficient similarity search and clustering of dense vectors).
  • Hybrid Retrieval: Combines BM25 (lexical retrieval) and FAISS for improved document retrieval using RRF and MMR techniques.
  • Answer Generation: Utilizes LLMs (using Ollama, OpenAI-compatible local LLM) to generate answers with citations from retrieved documents.
  • Self-Correction Mechanism: Implements a feedback loop to refine answers based on user input (WIP).
  • Guardrails: Applies regex-based filters to ensure generated content adheres to safety and compliance standards on input/output.
