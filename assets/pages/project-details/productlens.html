ProductLens

Agentic AI demo: LM-driven product comparison tool that ranks products based on a user's priorities.

ProductLens is a lightweight, tool-enabled “agentic” workflow that helps a user compare products in a structured, decision-ready way. It takes a natural-language request (e.g., “best laptop for battery + coding under $1,200”), plans what to research, gathers info in parallel, then returns a comparison with tradeoffs and a recommendation.

What it demonstrates

  • Planning from user intent: turns a messy request into clear evaluation criteria.
  • Parallel research: spawns one researcher per product to speed up gathering.
  • Explainable comparison: normalizes findings into consistent criteria + tradeoffs.
  • Light tool use: web search / browsing as an input to the comparison.
  • Simple UI: Gradio chat-style interface for quick demos.

How it works (high level)

  1. Comparison Manager (Orchestrator) receives the user request and coordinates the workflow.
  2. Planner Agent extracts priorities, selects candidate products, and outputs a structured plan.
  3. Research Agents (Parallel) each research one product using the plan’s criteria.
  4. Comparator / Decision Agent scores options, highlights tradeoffs, and recommends a best fit.

Tech stack

  • LLMs: local (Ollama) and/or cloud (OpenAI/OpenRouter) clients
  • Agent framework: OpenAI Agents SDK (tool calls + traces)
  • Tooling: Web search tool (or equivalent HTTP-based search)
  • UI: Gradio
  • Runtime: Python

Design notes

  • Focused agents: each agent does one job (plan, research, compare) to reduce confusion.
  • Structured intermediate outputs: planner output is used as the “contract” for research/comparison.
  • Decision-ready output: recommendation includes “why”, not just “what”.

Limitations (honest demo scope)

  • Not a full price-tracking system; results depend on what sources are found during search.
  • Not production-hardened (no long-term storage, ranking audits, or robust source filtering).
  • Evaluation is qualitative; a next step would be adding repeatable test queries and scoring.
